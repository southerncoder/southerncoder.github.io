Privacy Tier Content Processing Demo

This page demonstrates the three-tier privacy model implemented for content management. Each tier has different visibility, indexing, and access control characteristics.

Tier 1: Public Content

Characteristics:

  • Publicly accessible to all visitors
  • Indexed by search engines
  • Eligible for GitHub Pages mirroring
  • Included in sitemap.xml
  • No authentication required

Tier 2: Semi-Private Content

Characteristics:

  • Not indexed by search engines (noindex meta tag)
  • Accessible via basic authentication or obscure URLs
  • Excluded from GitHub Pages mirror
  • Not included in sitemap.xml
  • Robots.txt disallow entries

Tier 3: Private Content

Characteristics:

  • Requires full authentication (redirects to app.southerncoding.dev)
  • Never indexed by search engines
  • Never mirrored to GitHub Pages
  • Not included in sitemap.xml
  • User-specific state and interactions

Implementation Summary

Key Features Implemented:

Content Processing:

  • Privacy tier filtering utilities
  • Content visibility validation
  • Build-time content processing
  • Runtime access control

SEO & Indexing:

  • Dynamic robots.txt generation
  • Sitemap with public content only
  • Meta tags based on visibility
  • Canonical URL handling

Access Control:

  • Basic authentication middleware
  • Full authentication integration
  • Obscure URL generation
  • Authentication redirects

GitHub Pages Mirror:

  • Filtered build process
  • Explicit allowlist validation
  • CI/CD integration ready
  • Content security validation

Generated API Endpoints

/robots.txt

Dynamically generated robots.txt with allow/disallow rules based on content privacy tiers

/sitemap.xml

XML sitemap containing only public (Tier 1) content for search engine indexing
